리스트 파싱 테스트

HTML 백엔드의 다양한 리스트 파싱 기능을 검증하는 샘플 문서입니다.

1. 순서 없는 리스트 (ul)

기본적인 bullet 리스트입니다.

  • 첫 번째 항목
  • 두 번째 항목
  • 세 번째 항목
  • 네 번째 항목

2. 순서 있는 리스트 (ol)

번호가 붙는 순서 리스트입니다.

  1. 문서 분석 단계
  2. 데이터 추출 단계
  3. 구조화 단계
  4. 검증 단계
  5. 저장 단계

3. start 속성이 있는 순서 리스트

특정 번호부터 시작하는 순서 리스트입니다.

앞 단계에서 이어지는 목록:

  1. 인덱싱 단계
  2. 검색 최적화 단계
  3. 배포 단계

4. 중첩 순서 없는 리스트

ul 안에 ul이 중첩된 리스트입니다.

  • 프론트엔드 기술
    • HTML/CSS
    • JavaScript
      • React
      • Vue.js
      • Angular
    • TypeScript
  • 백엔드 기술
    • Python
      • FastAPI
      • Django
    • Go
    • Rust
  • 인프라
    • Docker
    • Kubernetes

5. 중첩 순서 있는 리스트

ol 안에 ol이 중첩된 단계별 절차입니다.

  1. 프로젝트 초기화
    1. 요구사항 분석
    2. 기술 스택 선정
    3. 환경 구성
  2. 개발 단계
    1. 핵심 기능 구현
    2. 테스트 작성
    3. 코드 리뷰
  3. 배포 단계
    1. 스테이징 환경 배포
    2. QA 검증
    3. 프로덕션 배포

6. 혼합 중첩 리스트 (ul + ol)

순서 없는 리스트 안에 순서 있는 리스트가 혼합된 구조입니다.

  • 문서 처리 파이프라인
    1. 입력 파일 로드
    2. 형식 감지
    3. 파싱 실행
  • 지원 파일 형식
    • PDF
    • DOCX
    • HTML
    • Markdown

7. 포맷팅이 포함된 리스트

리스트 항목 안에 텍스트 포맷팅이 포함된 경우입니다.

  • 중요: 이 항목은 굵은 텍스트로 강조되었습니다.
  • 참고: 이 항목은 기울임체로 표시됩니다.
  • 코드 예시: import docling
  • 링크가 포함된 항목: GitHub 저장소를 참고하세요.
  • 경고: 굵은 텍스트와 기울임체가 혼합된 항목입니다.

8. 리스트와 단락 혼합

각 리스트 항목이 여러 문장으로 구성된 복합 구조입니다.

  • 첫 번째 항목의 첫 번째 단락입니다. 항목 내에 단락이 포함될 수 있습니다.

    첫 번째 항목의 두 번째 단락입니다. 이 구조는 복잡한 문서에서 자주 사용됩니다.

  • 두 번째 항목입니다. 단일 단락으로만 구성된 항목입니다.

  • 세 번째 항목의 설명입니다.

    추가 정보가 담긴 단락입니다. 리스트 항목 내 단락은 올바르게 파싱되어야 합니다.

9. 정의 목록 (dl)

용어와 정의를 나타내는 dl 리스트입니다.

HTML
HyperText Markup Language의 약자로, 웹 페이지의 구조를 정의하는 마크업 언어입니다.
CSS
Cascading Style Sheets의 약자로, HTML 요소의 스타일을 정의하는 언어입니다.
JavaScript
웹 브라우저에서 동적 기능을 구현하는 프로그래밍 언어입니다.
